# 09 — Sample Communications

**Priority:** Recommended. Essential for getting written voice right.

## What to upload here

Real communications that have been sent. These train voice across different contexts.

### Press / media
- [ ] Recent press releases (FID, commercial milestones, project progress, partnerships)
- [ ] Media advisories
- [ ] Statements issued to media
- [ ] Media Q&A / reactive responses

### Internal
- [ ] All-hands emails from CEO / executives
- [ ] Leadership announcements (exec hires, promotions, departures)
- [ ] Organizational changes / restructuring comms
- [ ] Company newsletters
- [ ] Benefits / HR communications
- [ ] Safety bulletins / stop-work comms
- [ ] Weekly / monthly project updates

### External (non-media)
- [ ] Customer letters / counterparty communications
- [ ] Investor letters / shareholder letters
- [ ] Analyst briefing notes
- [ ] Community announcements
- [ ] Regulatory stakeholder letters (non-filing)
- [ ] Vendor / supplier comms

### Digital
- [ ] Approved LinkedIn post examples (corporate + exec)
- [ ] Approved Twitter/X post examples
- [ ] Website news / blog post examples

## File types expected

`.docx`, `.pdf`, `.msg` / `.eml` (email exports), `.txt`, `.html`

## What Claude will extract

- Subject line conventions (email)
- Headline conventions (press release)
- Opening paragraph structure (pyramid? anecdotal? dateline?)
- Who signs what (CEO? CFO? comms team? unsigned?)
- Approved signoffs ("Sincerely," vs. "Regards," vs. none)
- Standard call-to-action phrasing
- Boilerplate / "about NextDecade" block (this is huge — exact wording)
- Approved quotes patterns (how exec quotes are structured)
- Escalation / urgency language conventions
- Tone shifts by audience (investor vs. community vs. regulator vs. employee)
- Bilingual patterns (EN + ES where applicable)

## Notes

- The "about NextDecade" boilerplate paragraph at the end of every press release — capture the current version exactly. Also capture historical versions if it's evolved.
- If there's a crisis / incident communication example (redacted), it's enormously useful for crisis skill development — even a holding-statement template.
